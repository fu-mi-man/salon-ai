# 実装ロードマップ

ソースコードを「生きた仕様書」として扱い，詳細設計書の作成を最小限にしながら動くものを早く届ける方針。

コアの価値フロー：

```text
オーナーがサロンボードの口コミをコピー
  → Web UIに貼り付け・スタッフ選択
  → Gemini（few-shot プロンプト）→ Supabase に履歴保存
  → オーナーが返信文をコピーしてサロンボードにペースト
```

## フェーズ1 MVP（口コミ返信管理）

### Step 1 — ダッシュボードUI（ハードコード）

`docs/02_specification/screens/01_dashboard.md` の新仕様に沿ってUIを組む。Supabase 不要・ダミーデータで完成させる。

- [ ] `/dashboard` にアクセスすると「生成エリア」と「生成履歴」が表示される
- [ ] 口コミ入力テキストエリアが高さ自動伸縮する（上限20行）
- [ ] スタッフ選択セレクトボックスが表示される（ダミーの一覧）
- [ ] 入力または選択が空のときは生成ボタンが非活性になる
- [ ] 履歴カードに 生成日時・担当スタッフ・返信文本文 が表示される
- [ ] カードのコピー・編集・削除ボタンが動作する
- [ ] 編集モードに切り替わり，テキストエリアと「AIで修正」UIが表示される
- [ ] コピーボタンを押すと「コピーしました」が2秒表示される
- [ ] 履歴が0件のとき空状態メッセージが表示される
- [ ] タブレット幅（768px〜）で2カラム表示される（左:生成エリア，右:履歴）

### Step 2 — Supabase Auth 導入

`docs/90_wiki/setup/05_supabase.md` の構成に従い，認証基盤を用意する。

- [ ] ログイン画面（email / password）が表示される
- [ ] 未認証状態で `/dashboard` を開くとログイン画面へリダイレクトされる
- [ ] ログイン後は `/dashboard` に遷移する
- [ ] 全テーブルに RLS を設定し，自身の `salon_id` のデータのみ読み書きできる
- [ ] パスワードリセットが Supabase 標準のメールフローで動作する

### Step 3 — スタッフ・トーン例文の管理

スタッフおよび口調学習用の返信例文をオーナーが登録できるようにする。

- [ ] スタッフ一覧画面でスタッフを追加・編集・削除できる
- [ ] スタッフごとに返信例文を複数件登録・並べ替え・削除できる
- [ ] ダッシュボードのスタッフ選択セレクトボックスに登録済みスタッフが反映される

### Step 4 — Server Actions（返信履歴の永続化）

Supabase から返信履歴を取得・保存・更新する。

- [ ] Supabase の返信履歴がカードに表示される（ダミーデータ不要）
- [ ] カードの削除が DB に反映される
- [ ] 編集モードで保存するとカード内容が DB に反映される
- [ ] 最終更新から30日経過したレコードが pg_cron で自動削除される

### Step 5 — Gemini 連携（返信文生成）

Server Action から Gemini API を呼び出して返信文を生成する。  
プロンプト仕様は `docs/02_specification/integrations/gemini.md` を参照。

- [ ] 口コミ本文とスタッフ選択を送信すると，Gemini が返信文を生成する
- [ ] 選択スタッフのトーン例文が few-shot としてプロンプトに組み込まれる
- [ ] 生成結果が履歴カードとして DB に保存され，画面先頭に表示される
- [ ] 生成中は生成ボタンがローディング状態になる
- [ ] 生成失敗時は生成エリア下部にエラーメッセージが表示される

### Step 6 — AI修正（編集モードでの再生成）

編集モード内で修正指示を与えて返信文を書き換える。

- [ ] 編集モードの修正指示欄に入力して「AIで修正」を押すと Gemini が再生成する
- [ ] 再生成中はテキストエリアと各ボタンが disabled になる
- [ ] 完了後にテキストエリアが新しい内容で更新される
- [ ] 修正失敗時は修正指示欄の下にエラーメッセージが表示される
