# 実装ロードマップ

ソースコードを「生きた仕様書」として扱い，詳細設計書の作成を最小限にしながら動くものを早く届ける方針。

コアの価値フロー：

```text
GAS → メール検知 → Gemini → Supabase → Web UI → オーナーがコピペ
```

## フェーズ1 MVP（口コミ返信管理）

### Step 1 — ダッシュボードUI（ハードコード）

- コンポーネント内にダミーデータをハードコード（Supabase不要）
- `/dashboard` ページ（返信一覧，pending/doneフィルター）
- カード内編集・再生成・コピーボタンなどのインタラクションも含めて完成させる

画面仕様は `docs/02_specification/screens/01_dashboard.md` を参照。

- [x] /dashboard にアクセスするとカード一覧が表示される
- [x] デフォルトは「未対応」タブが選択されている
- [x] タブを切り替えると表示されるカードが絞り込まれる
- [x] カードに日時・ステータス・返信文・トーンが表示される
- [x] コピーボタンを押すと「コピーしました」が2秒表示される
- [x] 編集するを押すとカード内で返信文を編集できる
- [x] 「対応済みにする」を押すと未対応タブからカードが消える
- [x] 再生成ボタンでカード内にトーン選択が展開する（API呼び出しは Step 6）
- [x] タブレット幅（768px〜）で2カラム表示される
- [x] カード0件のとき空状態メッセージが表示される
- [ ] 読み込み中はスケルトンが表示される

### Step 2 — Server Actions（DB連携）

- Supabaseから返信一覧を取得する
- ステータス更新（pending → done）
- 返信テキスト編集・保存

- [ ] Supabase の返信一覧がカードに表示される（ダミーデータ不要）
- [ ] 「対応済みにする」を押すとDBのステータスが更新される
- [ ] ページをリロードしてもステータスが保持されている

### Step 3 — GAS: メール解析

- GmailからHPB口コミ通知メールを取得する
- レビュー内容をパースする
- `console.log` で動作確認（Gemini/Supabase不要）

仕様は `docs/02_specification/integrations/gas.md` を参照。

- [ ] HPB口コミ通知メールを受信すると GAS がトリガーされる
- [ ] メール本文から口コミ内容が正しくパースされる（console.log で確認）

### Step 4 — GAS: Gemini連携

- Gemini APIを叩いて返信文を生成する
- プロンプトは `gas/src/prompts/reply.ts` に切り出す

プロンプト仕様は `docs/02_specification/integrations/gemini.md` を参照。

- [ ] パースした口コミ内容から Gemini が返信文を生成する
- [ ] 生成された返信文が console.log で確認できる

### Step 5 — GAS: Supabase書き込み

- 生成した返信文をSupabase REST APIにPOSTする
- Web UIで表示されることを確認する（E2E初回）

- [ ] GAS が生成した返信文を Supabase に保存する
- [ ] Web UI のカード一覧に新しい返信文が表示される（E2E確認）

### Step 6 — UI: トーン変更・再生成

- Server ActionからGemini APIを呼ぶ
- カード内で「丁寧」「カジュアル」「簡潔」ボタンを出して再生成する

- [ ] カード内でトーンを選択すると新しい返信文が生成される
- [ ] 生成中はカード内がローディング状態になる
- [ ] 生成完了後にカードの返信文が更新される
- [ ] 生成失敗時はカード内にエラーメッセージが表示される
