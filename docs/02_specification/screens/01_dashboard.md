# ダッシュボード画面

**ルーティング**: `/dashboard`  
**対象ユーザー**: オーナー  
**主な目的**: 生成された返信文を確認・編集・コピーする  

## シナリオ

朝，オーナーがダッシュボードを開く  
→ 未対応の返信文が N 件表示されている  
→ 返信文を読んで問題なければコピーする  
→ HPBのサロンボードに貼り付ける  
→「対応済みにする」を押す → カードが一覧から消える  
→ 文章が気に入らなければ再生成ボタンでトーンを変えて作り直す  

## 画面構成

```
ヘッダー
  └ サービス名

フィルタータブ
  └ すべて / 未対応 / 対応済み

返信文カード一覧
  └ 返信文カード × N
```

## フィルタータブ

| タブ | 表示対象 |
|---|---|
| すべて | 全ステータス |
| 未対応 | pending |
| 対応済み | done |

デフォルトは「未対応」タブを選択した状態で表示する。

## 返信文カード

1件の返信文を1枚のカードで表示する。

### 表示項目

| 項目 | 内容 |
|---|---|
| 生成日時 | 例: 2026/04/03 08:00 |
| ステータスバッジ | pending →「未対応」/ done →「対応済み」|
| 返信文本文 | 生成されたテキスト全文 |
| トーン | 生成時のプリセット（丁寧 / カジュアル / 簡潔） |

### 操作

| 操作 | 挙動 |
|---|---|
| コピーボタン | 返信文をクリップボードにコピーする。コピー後はボタンのラベルが「コピーしました」に変わり2秒後に戻る |
| 編集するボタン | カード内で返信文をインライン編集する |
| 対応済みにするボタン | 確認後にステータスをdoneに更新する。「未対応」タブ表示中はカードが一覧から消える |
| 再生成ボタン | カード内でトーン選択ボタンを展開する |

### 再生成

「再生成する」ボタンを押すとカード内にトーン選択ボタンを展開する。

```
[キャンセル] [丁寧]
[カジュアル] [簡潔]
```

トーンボタンを押すとGemini APIを呼び出し、返信文を上書き保存する。  
生成中は対象カード内をローディング状態にする。

## 状態

| 状態 | 表示 |
|---|---|
| 読み込み中 | カードのスケルトンローダーを表示する |
| 返信文が0件 | 「対応が必要な口コミはありません」と表示する |
| コピー成功 | ボタンラベルを「コピーしました」に変更する（2秒） |
| 再生成中 | カード内の再生成UIをローディング状態にする |
| 再生成成功 | カードの返信文を更新する |
| 再生成失敗 | カード内にエラーメッセージを表示する |

## 備考

- SPファースト。カードは縦1列で表示する。タブレット幅（768px〜）では2カラム表示
- フェーズ1は認証なし。URLを知っていれば誰でもアクセスできる
- サロンボードのURLはカードに含めない（HPBへの自動遷移はしない）

## フェーズ設計メモ

### フェーズ1の制約

口コミ本文・投稿者情報を保存しないため（個人情報リスク回避），
カードには返信文・日時・ステータスのみ表示する。  
「誰の口コミへの返信か」という文脈はフェーズ1では持てない。

### フェーズ2の設計方向

口コミ本文・評価・投稿者名を保存できるようになる。設計はマスター・詳細型に移行する。

（PC・タブレット）左ペインに口コミ一覧（投稿者名・評価・日時），右ペインに選択した口コミの本文 + AI返信文 + 操作ボタン  
（モバイル）カードタップで返信文を展開する形式

### 実装上の注意（フェーズ1 → 2 の移行を見越して）

- ReplyCard コンポーネントは「口コミ情報」と「返信文情報」を明確に分離した props 設計にする
- レイアウトはフェーズ2でマスター・詳細に組み替えやすい構造にする
