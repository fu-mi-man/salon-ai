# ダッシュボード画面

**ルーティング**: `/dashboard`  
**対象ユーザー**: オーナー  
**主な目的**: 口コミを貼り付けて返信文を生成し，コピーする  

## シナリオ

オーナーがサロンボードで口コミを確認する  
→ 返信したい口コミ本文をコピーする  
→ ダッシュボードを開き，口コミ本文を貼り付ける  
→ 担当スタッフをセレクトボックスで選択する  
→「返信文を生成」を押す  
→ 生成された返信文が自動保存され，履歴の先頭に表示される  
→ コピーボタンを押してサロンボードに貼り付ける  

## 画面構成

```
ヘッダー
  └ サービス名

── 返信文を生成 ──────────────────
口コミ入力エリア（テキストエリア）
スタッフ選択（セレクトボックス） ＋ 生成ボタン

── 生成履歴 ──────────────────────
返信文カード × N
もっと見るボタン
```

## 生成エリア

### 口コミ入力

サロンボードからコピーした口コミ本文を貼り付けるテキストエリア。

- 入力内容に応じて高さが自動伸縮する（上限20行）
- 入力内容はDBに保存しない（生成時のみ使用）
- 生成後も入力内容はクリアしない（続けて別スタッフで生成できるように）

### スタッフ選択・生成

| 要素 | 内容 |
|---|---|
| セレクトボックス | 登録済みスタッフの一覧から担当者を選択する |
| 生成ボタン | 押下でGemini APIを呼び出し，選択スタッフの口調で返信文を生成する |

口コミ本文が空，またはスタッフが未選択の場合は生成ボタンを非活性にする。  
生成中はボタンをローディング状態にする。

## 生成履歴

### 返信文カード

1件の返信文を1枚のカードで表示する。  
生成日時の新しい順に並べる。

| 項目 | 内容 |
|---|---|
| 生成日時 | 例: 2026/04/03 08:00 |
| 担当スタッフ | 生成時に選択したスタッフ名 |
| 返信文本文 | 生成されたテキスト全文 |

### カード操作

通常時はカード下部に3ボタンを表示する。

| 操作 | 挙動 |
|---|---|
| コピー | 返信文をクリップボードにコピーする。コピー後はボタンのラベルが「コピーしました」に変わり2秒後に戻る |
| 編集 | カード内を編集モードに切り替える（後述） |
| 削除 | 確認後にカードを削除する |

### 編集モード

「編集」を押すとカード内が編集モードに切り替わる。

```
┌───────────────────────────────────┐
│ [返信文テキストエリア（編集可）]   │
│                                   │
├───────────────────────────────────┤
│ AIに修正指示: [_______________]   │
│                      [AIで修正]   │
├───────────────────────────────────┤
│ [キャンセル]           [保存]     │
└───────────────────────────────────┘
```

- テキストエリアを直接編集できる
- 修正指示欄に「もう少し短くして」などを入力して「AIで修正」を押すと，Gemini APIが既存の返信文と指示を踏まえて再生成し，テキストエリアを上書きする
- 「保存」を押すとテキストエリアの内容でカードを更新する

#### AI修正中のUI

「AIで修正」押下から結果返却まで：

- テキストエリアをdisabled（薄くグレーアウト）にする
- 「AIで修正」ボタンをスピナー＋「修正中...」に変える
- 「キャンセル」「保存」ボタンをdisabledにする
- 完了後，テキストエリアを新しい内容で更新し，全ボタンを通常状態に戻す

### 表示件数・もっと見る

- 初期表示は10件
- 「もっと見る」ボタンを押すたびに10件ずつ追加表示する
- 全件表示済みの場合はボタンを非表示にする

## 状態

| 状態 | 表示 |
|---|---|
| 生成中 | 生成ボタンをローディング状態にする |
| 生成成功 | 生成されたカードを履歴の先頭に追加する |
| 生成失敗 | 生成エリア下部にエラーメッセージを表示する |
| 履歴が0件 | 「まだ生成した返信文はありません」と表示する |
| コピー成功 | ボタンラベルを「コピーしました」に変更する（2秒） |
| AI修正中 | 編集モードのUI全体をローディング状態にする（上述） |
| AI修正失敗 | 修正指示欄の下にエラーメッセージを表示する |

## 備考

- SPファースト。生成エリア→履歴の順に縦並び。タブレット幅（768px〜）では2カラム表示（左: 生成エリア，右: 履歴）
- Supabase Auth でログインが必要。未認証の場合はログイン画面へリダイレクトする
- サロンボードのURLはカードに含めない（HPBへの自動遷移はしない）
- 履歴は最終更新から30日後に自動削除される
